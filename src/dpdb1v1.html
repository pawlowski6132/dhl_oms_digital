    Safety

    Required:

    • Safety cross must be used at the top, and displays the current month.
      • Tool tip within the safety cross to allow user to see the incident.
    • Days Safe is to be below the Safety Cross.
    • Safety Message to be below Days Safe.

    Safety Cross

    • Visual: Synoptic Panel
    • Background SVG Image: (link)
    • Tooltips contain Incident description, type, and created by
    • Colors
      • Green (Safe) -  #5D8158
      • Yellow (Near Miss/Non recordable incident) - #FFE066
      • Red (Recordable incident) - #E56970
    • Logics query below
      • Join Incident Master table with Personal Injury table
        • Personal injury table has the Recordable Incident column
    • Data set up
      • Pull in Logics Safety data using query below
      • Create reference of Date_Dim table and use Full_Dt and DoM_Nbr
      • Merge Logics Safety table into Dates Table on Full_Dt = Date of Occurence
      • Create conditional column that tags safe days as 0, non recordable incident days as 1, and recordable incidents as 2
      • Set up visual as shown below

    Tooltip:

    • The user should be able to hover a day that an incident occurred and see the detail associated with that incident.
    • Minimum Data Pieces
      • Site Location (if a network board).
      • Status
      • Description

    SQL Query (Replace SMRU with your site)

    SELECT

          a.[Incident_No]

          ,a.[Sub_Incident_Number]

          ,b.[Recordable_Injury]

          ,a.[Person_Responsible_Master_Incident]

          ,a.[Date_and_Time_of_Occurrence]

          ,a.[Date_Reported_to_Supervisor]

          ,a.[DHL_Date_Created]

          ,a.[Created_By]

          ,a.[Location]

          ,a.[Incident_Description]

          ,a.[Description_Of_Incident_Location]

          ,a.[Sub_Incident_Type]

          ,a.[Date_of_Occurrence]

          ,a.[DWH_INSERT_DATE]

      FROM [logics_final].[logics_Incident_1396_Mgmt_Master_final]a

      Left Join

           [logics_final].[logics_Incident_3333_Personal_Injury_final]b

              ON a.[Sub_Incident_Number] = b.[Sub_Incident_Number]

      where a.location like '%SMRU0840%'

    Quality

    Required

    • Bar chart with goal line 
    • Yesterday's value in the top right of chart

    Optional

    • Optional #1:  Summary Chart below the Required chart.
      • Purpose:  Interactive summary chart of categories that count against our goal.
      • Suggested time period of information:  Rolling 30 days
      • Functionality:  
        • Ability to click on a bar in the Required chart to view the number of occurrences of each category that occurred on that selected day.
      • Bar Direction:  Vertical or Horizontal
      • Tool tip:  Allow user to hoover over any selected bar to see the summary information.
      • Suggested Use:  When few instances occur on a daily basis. 
    • Optional #2:  Stacked bar chart below the required bar chart.
      • Purpose:  Interactive breakdown chart of items that went against our goal.
      • Suggested time period of information:  This should equal the number of days/bars that are in the Required chart. 
      • Bar Direction:  Vertical.
      • Tool tip:  Allow user to hoover over any selected bar to see the summary information.
      • Functionality:  Provides a breakdown of what occurred for each day and enables the ability to highlight a particular day of focus by selecting a day from the Required chart
